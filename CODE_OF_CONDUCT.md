# Code of Conduct

There is one developer here. Whatever community this project has amounts to the
occasional issue and whoever forked it last week. A document that assigns
responsibilities to "community leaders" and lays out a four-rung enforcement
ladder is describing a project that does not exist.

So, the short version.

Be decent. If you turn up in an issue, assume the person on the other end is
busy, working from partial information, and trying. That covers roughly
everything: no harassment, no slurs, no going after someone's identity, no
grinding personal axes in a bug report about GeoTIFF overviews.

If you are not decent, I will delete the comment and block you. That is the
entire enforcement apparatus. There is no committee, no appeals process, and no
promised response time, because there is nobody to staff them. Pretending
otherwise would just be a nicer-sounding lie.

This applies wherever the project is the subject — issues, pull requests,
discussions, or someone bringing it up elsewhere.

If you need to raise something privately, email mwkorver@gmail.com. Security
problems go through [SECURITY.md](SECURITY.md) instead, which is a real process
with a real inbox.

Worth repeating from [CONTRIBUTING.md](CONTRIBUTING.md): this is a reference
implementation, not a maintained project. Pull requests are not reviewed. Fork
it and make it yours — that is the intended path, and it needs no permission
from me.
